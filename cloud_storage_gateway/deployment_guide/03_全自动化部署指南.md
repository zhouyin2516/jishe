# 全自动脱壳部署指南 (One-Click Python Deployment)

当你新买了一台任意厂商提供的全新纯净版 Ubuntu 服务器后，你可以完全不需要人工 SSH 上去敲键盘装环境。这篇指南向你演示如何通过我们专门准备好的纯 Python 部署脚本完成服务发布和运维。

## 执行思路
在本地机器（比如你个人 Windows 笔记本）利用 `paramiko` 这个强大的 Python SSH 库。
我们通过编写脚本：
1. 自动注入 apt-get 更新。
2. 自动拉设隔离运行的 venv（独立虚拟 Python 环境），彻底不污染主机的环境变量。
3. 自动向 `/opt/cloud_storage_gateway/` SCP 传输所有的最新的工程源码。
4. 自动覆盖真正的生产级保护配置文件 `.env`。
5. 自动用守护模式（nohup 或者 systemd）在后台激起 Uvicorn 服务进程。

## 准备文件
打开本文件夹下的 `auto_deploy_template.py`（这个是我根据之前的联调精炼总结出的防阻塞挂死最终版本）。

## 配置你的专属参数与区域设定

当团队中其他同学去接手一台全新地区的新机器部署时，**必须且仅需**修改脚本开头的这几个全局变量：

```python
# --- 1. 服务器硬件访问信息 ---
HOST = '新服务器的公网或内网IP地址'
PASSWORD = '你的SSH密码' # 或配置SSH密钥连接

# --- 2. 联邦组密钥与云底层参数 (重点) ---
PRODUCTION_ENV_CONFIG = \"\"\"
# 你的 Java 后端想要调用该网关必须带上的防黑客密钥 (自定义)
INTERNAL_API_KEY=修改为强密码（对接Java后端专用）

# 你在华为云新开通的真实 OBS 桶的名称
OBS_BUCKET=medical-model-data

# 桶所在的节点域名。注意：如果是在贵阳，就是 obs.cn-southwest-2.myhuaweicloud.com；北京则是 obs.cn-north-4...
OBS_ENDPOINT=obs.cn-southwest-2.myhuaweicloud.com

# IAM 代理元数据的地区代号。必须和你的 ECS 所处的物理区域一致！(比如 cn-southwest-2)
IAM_REGION=cn-southwest-2
USE_ECS_AGENCY=True
\"\"\"
```

> [!IMPORTANT]
> **关键安全说明 (鉴权一致性)**：
> 部署脚本中设置的 `INTERNAL_API_KEY` 直接决定了网关的访问密码。
> - 在后续进行 API 调用（如 Postman 或 cURL 测试）时，HTTP Header 中的 `X-Internal-Token` **必须**与此处设置的值完全一致。
> - 如果此处修改为强密码，请务必同步更新所有的测试脚本和业务调用方的 Header 参数，否则会触发 `403 Forbidden` 报错。

## 执行发布


确保你的执行主机能够连接互联网，并且安装了**基础库**：
```powershell
pip install paramiko scp
```

随后直接执行：
```powershell
python auto_deploy_template.py
```
等待大约三分钟。全自动脚手架就会把原本空白的服务器变成一个稳定高性能能够吐出预签名大文件 URL 的独立微服务了。此时你就可以本地跑下面的指令验证：

```powershell
curl http://{HOST}:8000/health
```
