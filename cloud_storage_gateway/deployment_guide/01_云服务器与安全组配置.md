# 云服务器与网络基建配置

在使用该云存储网关微服务前，无论你购买的是华为云、阿里云还是任意提供商的 Linux 服务器，都需要进行标准化的基础设施配置。强烈建议操作系统采用 **Ubuntu 22.04 LTS 或 Ubuntu 24.04**。

## 1. 端口放行与安全组 (Security Group)
微服务默认运行在 `8000` 端口。这也就意味着如果云端机器的防火墙不向公网开放 8000 端口，任何对于网关的公网访问都会被立即丢弃（抛出超时或 TCP 连接重置错误）。

**操作路径示例（以华为云为例）：**
1. 登录云服务器控制台，点击对应 ECS 实例的“安全组”。
2. 点击“配置规则” -> “入方向规则” -> “添加规则”。
3. 协议端口配置协议为 `TCP`，端口为 `8000`。
4. **【极度强调：安全来源设限】** 来源地址（Source IP）强烈建议**不要**选择 `0.0.0.0/0`（这会向全世界黑客敞开大门）。**请填入你们联邦学习后端 Java 业务主服务器的真实固定公网 IP 或同一 VPC 内网 IP（如 `192.168.0.0/16`）**，这样网关将从物理底层上彻底隐身！
5. 保存并生效。

## 2. 检查网络内连通行
如果你通过内网或者特定跳板机访问：
确保服务器内部的 UFW / iptables 也没有误挡端口。Ubuntu 环境下可以通过以下命令验证：
```bash
sudo ufw status
# 如果为 active，则需：
sudo ufw allow 8000/tcp
```
